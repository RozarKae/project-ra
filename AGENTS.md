# Project RA — AI Development Instructions

## Project Identity

Project RA is a premium cinematic wedding invitation website.

It is not intended to be a generic event website.

The experience should feel emotional, elegant, luxurious, and cinematic from beginning to end.

Every design and coding decision must support that vision.

---

# Primary Objective

Create one of the highest quality wedding invitation experiences possible using modern frontend technologies.

The website should feel closer to a luxury brand landing page than a traditional wedding invitation.

# UI Rules

Always preserve:

Elegant typography

Large breathing spaces

Gold accents

Emerald secondary accents

Dark premium background

Glass morphism

Soft shadows

Rounded corners

Never introduce random colors.

Never introduce inconsistent spacing.

Never use loud animations.

---

# Animation Rules

Animations must always feel cinematic.

Preferred:

GSAP

Slow easing

Stagger animations

Parallax

Soft fade

Scale

Opacity

Never use:

Flashy effects

Excessive bouncing

Random rotations

Unnecessary spinning

Annoying infinite loops

---

# Coding Standards

Always write:

Readable code

Reusable code

Modular code

Semantic HTML

Organized CSS

Small JavaScript functions

Avoid duplication.

Avoid magic numbers.

Use descriptive variable names.

Comment only where necessary.

---

# CSS Standards

Use CSS variables.

Prefer reusable utility classes.

Group related styles.

Keep consistent spacing.

Avoid unnecessary specificity.

Never duplicate existing styles.

---

# JavaScript Standards

Avoid global pollution.

Reuse functions.

Cache DOM elements.

Use modern JavaScript.

Handle errors gracefully.

Never create duplicate listeners.

---
